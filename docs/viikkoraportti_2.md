# Viikkoraportti 2
Viikko meni shakin peruslogiikan ja ensimmäisten testien ohjelmointiin. Perusteet ovat lähellä valmiita, logiikasta puuttuu vielä liukuvat nappulat (torni, lähetti, kuningatar) ja shakin tarkistus. 
Liukuviin nappuloihin suunnittelen joko normi silmukan käyttämistä alkuun tai paljon nopeampia magic bitboardeja, jos se ei ole liian vaikeaa.

Viikolla opin paljon bitboardeista ja niiden välisistä vuorovaikutuksista. Myös koodin ylläpitäminen suhteellisen nopeana tulevaisuutta varten vaati jonkin verran vaivaa. 
Ensi viikolla suunnitelmana on päästä aloittamaan itse algoritmin koodaamista, eli shakin ydinlogiikka pitää saada viimeisteltyä ja koodia optimoitua enemmän. Liikefunktio sen sijaan, että se kertoo sallitut liikkeet yhdestä ruudusta palauttaa sallitut liikkeet kaikista ruuduista (lasketaan jokaiselle ruudulle oma liikebitboard) joka tulevaisuudessa nopeuttaa algoritmia huomattavasti.

### Käytetyt tunnit
Tiistai 17.3 2h - Bitboardit, pelilaudan visualisointi, syötteen muuntaminen koordinaateiksi.

Keskiviikko 1h 18.3 - Projektin rakenteen organisointi, optimointi, oikea liikkuminen.

Torstai 19.3 4h - Yleinen refaktorointi, nopeampi liikkuminen, ensimmäiset säännöt, testit, GitHub CI.
